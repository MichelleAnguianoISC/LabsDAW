Preguntas a responder

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
      • Para ejecutar la validación por parte del cliente siempre debemos recurrir a scripts, habitualmente en lenguaje JavaScript o emplear alguna librería para verificar que las entradas son del tipo correcto y que estén completas con datos, antes de que el formulario sea enviado. Esto facilita la programación y mejora de la interfaz del usuario.[1]
  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
      • Existen diferentes maneras de hacerlo, la más común es desactivando JavaScript. Otra forma es introducir un valor benigno en el campo de entrada en el navegador, interceptar el envío validado con su proxy y modificar los datos a su valor deseado.[2]
  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
      • Como usuario se tiene una mejor experiencia con la interfaz, ya que no tendría problemas para ingresar los datos que le solicitan y como programador, es más fácil a la hora de enviar la información al servidor, para no recibir ningun error de ese lado, es mejor que ocurra en JavaScript que en la comunicación.
